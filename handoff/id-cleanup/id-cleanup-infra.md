## 2. 배경 / 공지 내용

monitoring-meta 기준 문서(ROADMAP 등)의 식별자 체계가 **운영 3종(T/D/ADR)으로 단순화**됐다(2026-06-11 결정).

- **G-n(게이트) 폐지 → D-n으로 흡수**: G-1→D-1, G-2→D-2, **G-3→D-8**, G-4→D-7, G-5→§5 완료 조건 7
- **DoD-n 폐지 → "§5 완료 조건 N"** (번호 1~7 보존)
- 읽는 법·신구 매핑: `../monitoring-meta/docs/phase1/ID-GLOSSARY.md`

## 3. infra에 미치는 영향 — **변경 0**

infra 내 ID 참조는 전부 주석(docker-compose.yml의 T1-1/G-3 후속 과제 주석, init 스크립트의 P-2/T3-5 주석 등)이라 동작 영향이 없고, **소급 수정하지 않는다**. 떠 있는 스택(T1-1 검증용)도 건드리지 않는다.

다만 주석을 읽을 때 해독 기준:
- 주석 속 `G-3` = 현 **D-8**(사이트별 운영 정보, 통합본 §13_open §A)
- 주석 속 `P-2` = T1-1 운영골격 슬라이스(일회성 ID, glossary 부록 D)

다음에 해당 파일을 *다른 사유로* 수정할 일이 생기면 그때 주석의 옛 ID를 새 표기로 같이 갱신해도 좋다(선택, 의무 아님).
