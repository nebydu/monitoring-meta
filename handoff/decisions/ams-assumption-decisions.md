# AMS 분석 가정 — 잠정 확정 누적 기록

> **성격**: 통합본 13장 §J 14항목의 **사용자 잠정 확정값·근거·날짜·재검증 대상**을 누적 기록하는 결정 자산. 프로세스 결정(2026-06-13, D1~D4)은 `handoff/ams-assumption-baseline/ams-assumption-baseline-000-decision-packet.md` §7·§9.
> **규칙**: ① 잠정 확정 시 통합본 해당 절 태그를 `[AMS 분석 가정 — 사용자 잠정 확정 YYYY-MM-DD, 팀 리뷰 재검증]`으로 전환하고 이 파일에 행 추가. ② 진행은 pull — 차단 시점별 묶음(아래 시점 컬럼). ③ **팀 리뷰 게이트 = 마이그레이션/ETL 설계 착수 전 + 늦어도 본개발 진입 전** — 게이트에서 이 파일 전체를 재검증한다.
> **④ 잠정 확정 행에는 「되돌릴 설계 범위」를 함께 적는다** — 그 가정 위에 어떤 설계/코드가 섰는지(반증 시 재작업 범위). 비어 있으면 "아직 설계 미착수"로 명시. *추적 의무이지, 설계를 어떻게 격리할지는 해당 구현 세션(T2-1 등)의 결정이다 — 여기서 격리 방식을 규정하지 않는다.*
> **⑤ 채널 부재 fallback(C1 보완)**: 팀 리뷰 채널이 끝내 마련되지 않은 채 게이트 시점(마이그레이션/ETL 설계 착수)이 임박하면 → **사용자 2차 재승인으로 팀 리뷰를 갈음**하고(이 파일에 재승인 날짜 기록), 그래도 불확실한 고위험 항목(역전 비용 높음)은 **DEFERRED 처리**해 해당 설계 착수를 미룬다. 미검증 잠정 확정이 무한 누적되는 것을 막는 종료 조건이다.
> **⑥ T2-1(Rule Engine) 게이트 기록 형식**: J-04·J-05·J-14를 잠정 확정할 때는 아래 표에 행을 추가하되 「잠정 확정값」칸에 **확정값 + 반증 시 되돌릴 설계 범위 + (있으면) 대체 설계 옵션**을 함께 적는다 — 별도 1페이지 문서는 만들지 않는다(확정 시점에 이 표로 충분).
