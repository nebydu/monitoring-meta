### 2.2 drift 분류

**기준 문서가 더 새로움 — 경로 stale(운영 위험 있음) + 호칭 stale(표기 정합)**

구 경로 `docs/통합본_v0_9.md`는 redirect tombstone stub으로 남아있어 파일 자체는 존재하지만 내용이 없으므로 형제 repo 에이전트가 이 경로를 직접 읽으면 빈 내용을 보게 된다. **경로 stale은 에이전트 동작(특히 analyzer/spec-guardian의 "통합본 읽기"에서 빈 파일을 참조하는 오동작)에 실질 위험을 가진다.**

호칭 stale("통합본 v0.9" 표기)은 표기 정합 문제이며 기능 위험은 낮다.

**발주 대기 분류 — 신규 drift 아님**

위 항목들은 `handoff/spec-backfill/` 3건으로 이미 발주됐다. 형제 repo 세션 미적용은 **"발주 대기 상태로 기록되는 것이 정상"** (배경 §1 기준)이며, 이 시점에서의 신규 drift가 아니다.

## 3. v0.11 backfill 내용 vs 형제 repo 코드 충돌 검사

v0.11은 "코드가 먼저 가 있었고 문서가 따라간 backfill"이므로 코드와 충돌이 없어야 정상이다.

| backfill 항목 | 기준 문서(v0.11) 내용 | 형제 repo 실제 상태 | 판정 |
|---|---|---|---|
| heartbeat protobuf 전환(ADR#2) | `otlp_proto` 완료, `otlp_json` 폐기 | `infra/otel-collector-config.yml:20` = `encoding: otlp_proto` — 실제 protobuf 운영 중 | **일치 — drift 없음** |
| command-topic 단일 물리 토픽(ADR#5) | `command-topic` = 단일 물리 토픽(zone 전개는 미래) | `infra/docker-compose.yml:64` = `for t in command-topic ...` — 단일 토픽 생성 중 | **일치 — drift 없음** |
| T4-1 완료(3토픽 물리=논리 일치) | `command-topic`, `audit-topic`, `heartbeats-topic` 물리=논리 | `infra/docker-compose.yml:64` = `command-topic`, `audit-topic`, `heartbeats-topic` 생성 | **일치 — drift 없음** |
| `job-results` 잔존(T4-2 미완) | `job-results` → `result-topic-job/log` 분리는 T4-2 잔여 | `infra/docker-compose.yml:64` = `job-results` 유지 | **일치 — drift 없음** |
| envelope.md 상위 근거 경로 | `docs/master-design.md`(통합본) | `docs/envelope.md:10` = `docs/master-design.md`(통합본) 이미 갱신됨 | **일치** |

**결론**: v0.11 backfill 내용과 형제 repo 코드 간 충돌하는 신규 drift **없음**. 예상대로 코드가 선행, 문서가 후행 정합이다.
