# D-2 escalation 타이밍 판단 메모

> **성격**: 이 메모는 D-2(β 모듈러모놀리스 유지 vs γ MSA 분리)를 *결정*하지 않는다. D-2가 통합본 §13_open §C 미결 Open이라 추측 금지(CLAUDE.md §2)다. 이 메모가 답하는 것은 **"D-2를 *언제* 사람에게 escalation해야 하는가"** 라는 *타이밍*뿐이다.
>
> **근거**: `handoff/phase1-critical-path-analysis.md`(§2 의존 그래프·§4 critical path·§5 레버리지) + ROADMAP §10~§17.
> **작성**: meta(오케스트레이터), 2026-06-07.

---

## 1. D-2가 *최초로* 막는 작업 (= 런웨이의 끝)

| 막는 대상 | 무엇이 막히나 | 근거 § |
|---|---|---|
| **T2-1 (Rule Engine)** | CP-1(코어 도메인 사슬)의 2단. `blocked_by = T1-3 일부(D-2)` — D-2가 안 풀리면 *착수 범위*가 모호 | §11 T2-1 / 분석 §4 CP-1 |
| **T1-3 (deployment 분리)** | status 자체가 `DECISION_REQUIRED` = D-2 그 자체 | §10 T1-3 |
| **D-6 (owner_repo)의 β/γ 의존분** | β/γ에 따라 달라지는 repo 배치만. 비의존분은 지금 확정 가능 | §17 D-6 |
| **DoD-4** | Phase 1 완료 하드 게이트(맨 끝) | §5 DoD-4 |

→ **실행 타임라인상 D-2 벽이 처음 닿는 지점 = T2-1(Rule Engine) 착수 시점.** 그 전까지는 D-2 없이 진행 가능하다. (T1-3는 "결정" 노드, DoD-4는 맨 끝 게이트라 *실행 흐름*에서 먼저 부딪히는 건 T2-1.)

## 2. 그 전까지 D-2 없이 가능한 작업 (= 런웨이 R)

분석 §3.2 READY + §6 실행 레인에서, **T2-1에 닿기 전까지** D-2와 무관하게 굴릴 수 있는 작업:

| 단계 | 작업 | D-2 무관 근거 |
|---|---|---|
| 즉시 | **T1-1 영속(site-중립 슬라이스)** PG/OS/Redis/MinIO 프로비저닝·연결 | 엔진은 β/γ 공통 self-host(`phase1-050-t1-1-datastore-slice.md`) |
| 즉시 | **T4-2** result 분리 / **T4-4** 메시지 키 / **T4-5** contract 문서 | T4-1(DONE) 후속, D-2 무관 |
| T1-1 후 | **T1-2** auth / **T1-4** Quartz JobStore / **T2-8** dedup / **T3-5** storage | 전부 T1-1 선행, D-2 무관 (§2.2) |
| 병행 | 결정 **D-5**(ADR 소속)·**D-3**(실행순서)·**D-6 비의존분** | 통합본 Open 아님, 지금 확정 가능 |

→ **런웨이 R = Track 1 코어(T1-1→T1-2/T1-4/T2-8/T3-5) + Track 4 잔여(T4-2/T4-4/T4-5) + 결정 3건.** 분량이 **크다**(다수 작업 단위, 수일~수주 규모). 즉 D-2 벽까지 *상당한 거리*가 있다.

## 3. 협의 입력 리드타임 L (외부 변수 — meta가 못 잼)

D-2는 통합본 §13_open §C의 **협의 입력 8개**(K8s 운영 인력·CI/CD 성숙도·팀 구조·운영 규모·Phase 1 일정 압박·서비스 독립 진화 요구·장애 격리 요구 등 — 권위 목록은 통합본 §C)가 모여야 결정 가능하다.

- 이 입력들은 **조직/운영 사실**이라 사람(외부 의사결정자)만 제공 가능 → **리드타임 L은 meta가 측정 불가**.
- L은 "8개 입력을 모아 β/γ를 판단하는 데 걸리는 시간"이며, 가용성에 따라 *며칠~수주*.

## 4. 결론 — 트리거 규칙

**"즉시 결정 강제"도 "무기한 방치"도 아니다.** *입력 요청*과 *결정 강제*를 분리한다:

1. **지금 (병행, 비용 0)**: **D-2 협의 입력 8개 수집을 *지금* 사람에게 요청**한다. 이건 escalation이되 "지금 β/γ를 정하라"가 아니라 **"§C 8개 입력을 모으기 시작하자"** 다. 실행 레인(T1-1 등)을 전혀 막지 않으므로 비용이 없다.
2. **결정 데드라인 = T2-1 착수 직전**. 런웨이 R(Track 1 코어 + Track 4 잔여)을 다 소진해 **T2-1에 닿기 전까지** β/γ가 정해지면 critical path에 지연 0.
3. **트리거 규칙**: `남은 런웨이 < L` 이면 **즉시 결정 강제로 격상**.
   - R(런웨이)은 **크고**(§2), L(리드타임)은 외부 변수다. 따라서 **현 시점 권장 = "입력 수집은 지금 시작, 결정 강제는 T2-1 게이트까지 유보"**.
   - 단 L이 예상보다 길어질 신호(입력 수집이 안 도는 등)가 보이면, T1-1 코어 진행 중이라도 **조기 결정 강제**로 올린다.

### 한 줄 결론
> **D-2는 "지금 *입력 수집* 요청(병행, 무비용)" + "결정은 T2-1 착수 전까지 유보". 런웨이가 크므로 *즉시 결정 강제*는 불필요하나, 입력 수집 리드타임이 런웨이를 넘기는 신호가 보이면 조기 격상.**

이전의 반사적 "D-2를 무조건 1순위로 즉시 escalation"은 *런웨이를 무시*한 과한 판단이었고, 본 메모로 **"입력은 지금, 결정은 T2-1 게이트"** 로 교정한다.

## 5. 사람 입력 대기

- **지금 요청**: D-2 협의 입력 8개(통합본 §C) 수집 착수 — 누가/언제까지 모을 수 있는지(=리드타임 L 추정).
- **유보**: β/γ 최종 결정 자체는 T2-1 착수 전까지. (L > 런웨이 신호 시 조기 격상.)
